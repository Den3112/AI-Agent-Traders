# Инструкции для AI-агентов (AI Trade Team)

Этот документ содержит операционные инструкции для управления проектом и разработки новых компонентов.

## 🧠 Паттерны мышления

- **Сначала архитектура**: Перед изменением кода проверь `ARCHITECTURE.md`.
- **Проверка на "хрупкость"**: Торговые системы не прощают ошибок. Всегда предусматривай `try-except` и логирование.
- **Минимизация галлюцинаций**: Если не уверен в параметрах API (например, Binance), используй `search_web` или читай документацию в `node_modules` (для JS).

## 🛠 Работа с Hardened Skills (Python)

Все навыки в `/skills/` должны следовать шаблону:
1. **Входные данные**: Принимаются как аргументы командной строки или JSON.
2. **Выходные данные**: Строго валидный JSON в `stdout`.
3. **Логирование**: Ошибки в `stderr`.
4. **Типизация**: Использование `typing` обязательно.

## 🎨 Гайд по Premium UI (Dashboard)

- **Цвета**: Не использовать чистый `#000` или `#fff`. Использовать глубокие темные тона (`#0a0a0b`) и мягкие акценты.
- **Интерактивность**: Каждый клик или наведение должно иметь визуальный отклик (transition: 0.2s).
- **Данные**: Использовать `Skeleton Loaders` для ожидания данных из торговых API.

## 🏁 Чек-лист перед завершением задачи

- [ ] Код проходит линтинг (`ruff` / `eslint`).
- [ ] Типы проверены.
- [ ] Переменные окружения добавлены в `.env.example`.
- [ ] Логика протестирована в `PAPER` режиме (если применимо).
- [ ] Ответ пользователю дан на **русском языке**.
